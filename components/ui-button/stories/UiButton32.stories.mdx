# UiButton32

Buttons allow users to take actions, and make choices. Simple example:

<Canvas>
  <UiButton32 variant="primaryDesign">Click me!</UiButton32>
</Canvas>

## Types

The component supports four types: `button`, `a`, `div` and `link`. By default, the `button` type is used.

### Button

By default, the component renders generic button:

<Canvas>
  <UiButton32 variant="primaryDesign">Default</UiButton32>
</Canvas>

You can provide `as="button"` explicitly:

<Canvas>
  <UiButton32 as="button" variant="primaryDesign">Button</UiButton32>
</Canvas>

### Anchor

You can render `a` element with providing `as="button"`:

<Canvas>
  <UiButton32 as="a" href="#" target="_blank" variant="primaryDesign">Anchor</UiButton32>
</Canvas>

### Div

In rare cases, you can render `div` element with providing `as="div"`:

<Canvas>
  <UiButton32 as="div" variant="primaryDesign">Div</UiButton32>
</Canvas>

### Link

You can use `as="link"` and provide `component` property to use `react-router`'s `Link` component:

<Canvas>
  <UiButton32 as="link" component={Link} target="_blank" to="#" variant="primaryDesign">Link</UiButton32>
</Canvas>

It can be useful, when you want to have `div` element with button appearance. For example, to render it inside
another `button` element.

## Options

The component supports a few options.

### `isDisabled`

This property allows to disable button:

<Canvas>
  <UiButton32 as="button" isDisabled variant="primaryDesign">Button</UiButton32>
  <UiButton32 as="a" href="#" isDisabled variant="primaryDesign">Anchor</UiButton32>
  <UiButton32 as="div" isDisabled variant="primaryDesign">Div</UiButton32>
  <UiButton32 as="link" component={Link} isDisabled to="#" variant="primaryDesign">Link</UiButton32>
</Canvas>

### `isFrozen`

This property allows to disable button, but keep enabled look and feel:

<Canvas>
  <UiButton32 as="button" isFrozen variant="primaryDesign">Button</UiButton32>
  <UiButton32 as="a" href="#" isFrozen variant="primaryDesign">Anchor</UiButton32>
  <UiButton32 as="div" isFrozen variant="primaryDesign">Div</UiButton32>
  <UiButton32 as="link" component={Link} isFrozen to="#" variant="primaryDesign">Link</UiButton32>
</Canvas>

### `isDisabled` and `isFrozen`

The `isDisabled` styles has priority over `isFrozen` when they're used together.

<Canvas>
  <UiButton32 as="button" isDisabled isFrozen variant="primaryDesign">Button</UiButton32>
  <UiButton32 as="a" isDisabled isFrozen href="#" variant="primaryDesign">Anchor</UiButton32>
  <UiButton32 as="div" isDisabled isFrozen variant="primaryDesign">Div</UiButton32>
  <UiButton32 as="link" component={Link} isDisabled isFrozen to="#" variant="primaryDesign">Link</UiButton32>
</Canvas>

### `icon`

You can provide an icon component:

<Canvas>
  <UiButton32 as="button" icon={AddIcon} variant="primaryDesign">Button</UiButton32>
  <UiButton32 as="a" icon={AddIcon} href="#" target="_blank" variant="primaryDesign">Anchor</UiButton32>
  <UiButton32 as="div" icon={AddIcon} variant="primaryDesign">Div</UiButton32>
  <UiButton32 as="link" icon={AddIcon} component={Link} target="_blank" to="#" variant="primaryDesign">Link</UiButton32>
</Canvas>

We recommend use icons with size of `16x16`.

An icon component should support `className` property.

## Attributes

All component types supports relevant HTML attributes, excluding `aria-disabled` and `disabled`. This attributes are
controlled by the component itself.

Also `href` attribute is restricted `link` type.

For example, you can provide `data-*` attribute for analytics purposes:

<Canvas>
  <UiButton32 as="button" data-track-id="button" variant="primaryDesign">Button</UiButton32>
  <UiButton32 as="a" data-track-id="anchor" href="#" target="_blank" variant="primaryDesign">Anchor</UiButton32>
  <UiButton32 as="div" data-track-id="div" variant="primaryDesign">Div</UiButton32>
  <UiButton32 as="link" data-track-id="link" component={Link} target="_blank" to="#" variant="primaryDesign">Link</UiButton32>
</Canvas>
