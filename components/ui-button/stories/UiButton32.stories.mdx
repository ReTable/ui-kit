# UiButton32

Buttons allow users to take actions, and make choices. Simple example:

<Canvas>
  <UiButton32 variant="primaryDesign">Clickable Button</UiButton32>
</Canvas>

## Demos

The component supports are many options.

### As

The components supports three modes: `button`, `link` and `visual`. By default, the `button` mode is used.

#### Button

By default, the component renders generic button:

<Canvas>
  <UiButton32 variant="primaryDesign">Default</UiButton32>
  <UiButton32 as="button"  variant="primaryDesign">Button</UiButton32>
</Canvas>

#### Link

You can use button as link providing `as="link"` property, and additionally `href` and optionally `target`:

<Canvas>
  <UiButton32 as="link" href="#" variant="primaryDesign">Link</UiButton32>
  <UiButton32 as="link" href="#" target="_blank" variant="primaryDesign">Link with target</UiButton32>
</Canvas>

#### Visual

You can render button as `div` element providing `as="visual"` for rare cases when you can't use `button` element:

<Canvas>
  <UiButton32 as="visual" variant="primaryDesign">Visual</UiButton32>
</Canvas>

### Variant

There are 6 variants:

<Canvas>
  <UiButton32 variant="primaryDesign">Primary Design</UiButton32>
  <UiButton32 variant="secondaryDesign">Secondary Design</UiButton32>
  <UiButton32 variant="primaryList">Primary List</UiButton32>
  <UiButton32 variant="secondaryList">Secondary List</UiButton32>
  <UiButton32 variant="dangerousList">Dangerous List</UiButton32>
  <UiButton32 variant="shadowList">Shadow List</UiButton32>
</Canvas>

### Disabled

All buttons supports `isDisabled` property:

<Canvas>
  <UiButton32 as="button" isDisabled variant="primaryDesign">Button</UiButton32>
  <UiButton32 as="link" isDisabled href="#" variant="primaryDesign">Link</UiButton32>
  <UiButton32 as="visual" isDisabled variant="primaryDesign">Visual</UiButton32>
</Canvas>

### Frozen

Along with `isDisabled` option, the component supports `isFrozen` option. In that case, button will have enabled visual
style, but will not react on user interactions.

<Canvas>
  <UiButton32 as="button" isFrozen variant="primaryDesign">Button</UiButton32>
  <UiButton32 as="link" isFrozen href="#" variant="primaryDesign">Link</UiButton32>
  <UiButton32 as="visual" isFrozen variant="primaryDesign">Visual</UiButton32>
</Canvas>

### Icon

You can provide an icon component:

<Canvas>
  <UiButton32 icon={AddIcon} variant="primaryDesign">Button</UiButton32>
  <UiButton32 icon={AddIcon} variant="secondaryDesign">Button</UiButton32>
  <UiButton32 icon={AddIcon} variant="primaryList">Button</UiButton32>
  <UiButton32 icon={AddIcon} variant="secondaryList">Button</UiButton32>
  <UiButton32 icon={AddIcon} variant="dangerousList">Button</UiButton32>
  <UiButton32 icon={AddIcon} variant="shadowList">Button</UiButton32>
</Canvas>

We recommend use icons with size of `16x16`.

An icon component should supports `className` property.

### Analytics

Also, the component supports the `trackId` property. It's adds `data-track-id` property to component for analytics
purposes.

<Canvas>
  <UiButton32 as="button" trackId="my-track-id" variant="primaryDesign">Tracked</UiButton32>
</Canvas>
