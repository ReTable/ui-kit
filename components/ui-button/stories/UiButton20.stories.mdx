# UiButton20

Buttons allow users to take actions, and make choices. Simple example:

<Canvas className={contractBackground}>
  <UiButton20 variant="contract">Clickable Button</UiButton20>
</Canvas>

## Demos

The component supports are many options.

### Type

The component supports three types: `button`, `link` and `visual`. By default, the `button` type is used.

#### Button

By default, the component renders generic button:

<Canvas className={contractBackground}>
  <UiButton20 variant="contract">Default</UiButton20>
  <UiButton20 type="button" variant="contract">Button</UiButton20>
</Canvas>

#### Link

You can use button as link providing `type="link"` property, and additionally `href`. Also, you can provide `rel` and `target`:

<Canvas className={contractBackground}>
  <UiButton20 href="#" type="link" variant="contract">Link</UiButton20>
  <UiButton20 href="#" rel="noreferrer noopener" target="_blank" type="link" variant="contract">Link with target and rel</UiButton20>
</Canvas>

#### Visual

You can render button as `div` element providing `type="visual"` for rare cases when you can't use `button` element:

<Canvas className={contractBackground}>
  <UiButton20 type="visual" variant="contract">Visual</UiButton20>
</Canvas>

### Variant

There is single variant:

<Canvas className={contractBackground}>
  <UiButton20 variant="contract">Contract</UiButton20>
</Canvas>

### Disabled

All buttons supports `isDisabled` property:

<Canvas className={contractBackground}>
  <UiButton20 isDisabled type="button" variant="contract">Button</UiButton20>
  <UiButton20 href="#" isDisabled type="link" variant="contract">Link</UiButton20>
  <UiButton20 isDisabled type="visual" variant="contract">Visual</UiButton20>
</Canvas>

### Frozen

Along with `isDisabled` option, the component supports `isFrozen` option. In that case, button will have enabled visual
style, but will not react on user interactions.

<Canvas className={contractBackground}>
  <UiButton20 isFrozen type="button" variant="contract">Button</UiButton20>
  <UiButton20 href="#" isFrozen type="link" variant="contract">Link</UiButton20>
  <UiButton20 isFrozen type="visual" variant="contract">Visual</UiButton20>
</Canvas>

### Disabled and frozen

The `isDisabled` has priority when you use `isDisabled` and `isFrozen` together.

<Canvas className={contractBackground}>
  <UiButton20 isDisabled isFrozen type="button" variant="contract">Button</UiButton20>
  <UiButton20 isDisabled isFrozen href="#" type="link" variant="contract">Link</UiButton20>
  <UiButton20 isDisabled isFrozen type="visual" variant="contract">Visual</UiButton20>
</Canvas>

### Icon

You can provide an icon component:

<Canvas className={contractBackground}>
  <UiButton20 icon={AddIcon} variant="contract">Button</UiButton20>
</Canvas>

We recommend use icons with size of `16x16`.

An icon component should support `className` property.

### Analytics

Also, the component supports the `trackId` property. It's adds `data-track-id` attribute to component for analytics
purposes.

<Canvas className={contractBackground}>
  <UiButton20 trackId="my-track-id" type="button" variant="contract">Tracked</UiButton20>
</Canvas>
